Introduction:


This week at ISSessions, in our last meeting before our annual CTF, I presented a quick overview of a recent vulnerability in a popular WordPress plugin called "Quiz and Survey Master" or "QSM" for short. This vulnerability caught my attention because it is an exploit that even an attacker with the lowest of permissions on a target site can manipulate to cause damage and access sensitive information. You can find the initial news article about this vulnerability on Infosecurity Magazine's website here, the initial vulnerability report from Patchstack here, and the video of me covering the vulnerability at our ISSessions meeting here.


Summary Of The Articles & Vulnerability:


Researchers at Patchstack have discovered a Medium to High severity SQL Injection vulnerability in versions 10.3.1 and below of the "Quiz and Survey Master" WordPress plugin, which has been installed on over 40,000 sites. The vulnerability, identified as CVE-2025-67987, was first discovered on November 21st 2025 and was patched on December 4th 2025, but was not publicly disclosed until January 28th 2026. This vulnerability allows attackers exploit it with the lowest of permissions (Subscriber role) to execute arbitrary SQL commands on the vulnerable site. This can lead to a wide range of potential impacts, including data theft, data manipulation, and even complete site takeover. The vulnerability is caused by a lack of proper input validation in the plugin's code, which allows attackers to inject malicious SQL commands into the plugin's database queries.


Technical Breakdown of the Vulnerability:


The main vulnerable snippet of code that leaves the plugin succeptable to SQL Injection resides in the "qsm_rest_get_question" function in the "php/rest-api.php" file of the plugin. The plugin works uder the assumption that the "is_linking" parameter value is an ID, but it fails to do any sort of validation or sanitization of the value before it is used directly in an SQL query (WHERE `question_id` IN (".$comma_seperated_ids.")"). Due to how the value is not confirmed to be a valid Integer ID, and it is autmatically concatinated into a SQL statement, a malicious actor can easily inject an arbitrary SQL statement alogside an abnormal value for the "is_linking" parameter to execute arbitrary SQL commands on the vulnerable site. For example, an attacker could set the "is_linking" parameter to a value like "1); DROP TABLE wp_users; --", which would cause the plugin to execute a SQL query that drops the "wp_users" table from the database, effectively deleting all user accounts on the site. This is just one example of how an attacker could exploit this vulnerability.



The vulnerability was patched in version 10.3.2 of the plugin, which was released in early December of 2025. The patch involved adding proper input validation and sanitization for the content of the "is_linking" parameter through the usage of the "intval()" function, which ensures that the value is indeed an integer before it is used in the SQL query. This patch effectively mitigates the vulnerability by removing the ability to input anything other than an integer in the field.
